- `네트워크 🔥`
  - 네트워크에서 패킷이란 무엇을 의미하나요?
    - 인터넷등 컴퓨터 네트워크를 통해 전송되는 데이터를 작은 단위로 나눈 것
  - 네트워크에서 처리량은 무엇을 의미하나요?
    - 특정 시간 동안 네트워크 링크를 통해 전송받은 데이터의 양
  - 지연시간이란 무엇인가요?
    - 데이터가 출발지에서 목적지까지 도할하는데 걸리는 시간
  - 처리량은 어떤 상황에 영향을 받나요?
    - 트래픽 집중, 사용자 수 증가, 대역폭 제한, 하드웨어 성능 등
  - 네트워크 병목현상은 무엇인가요?
    - 네트워크에서 데이터 전송 속도가 느려지는 현상
  - TCP/IP 가 무엇인가요? 각 계층에 대해서 설명할 수 있나요?
    - TCP/IP는 인터넷 프로토콜 스위트로, 네트워크 통신을 위한 표준 프로토콜 모음입니다.
      - 네트워크 인터페이스 계층: 물리적 네트워크 연결과 데이터 전송을 담당합니다.
      - 인터넷 계층: IP 주소를 사용하여 패킷을 목적지까지 라우팅합니다.
      - 전송 계층: 데이터의 신뢰성 있는 전송을 보장하며, TCP와 UDP가 포함됩니다.
      - 응용 계층: 사용자와 직접 상호작용하는 애플리케이션 프로토콜을 다룹니다. (예: HTTP, FTP)
      - 응용계층 -> 전송계층 -> 인터넷계층 -> 네트워크인터페이스계층
  - 전송 계층의 대표적인 방식은 무엇이 있고 어떤 차이가 있나요? (키워드: TCP, UDP)
    - TCP(Transmission Control Protocol): 연결 지향적이며, 신뢰성 있는 데이터 전송을 보장합니다. 데이터가 순서대로 도착하고 손실된 패킷을 재전송합니다.
    - UDP(User Datagram Protocol): 비연결 지향적이며, 빠른 전송을 위해 신뢰성을 보장하지 않습니다. 실시간 애플리케이션에 적합합니다.
  - 계층 간 데이터 송수신 과정에 대해 설명할 수 있나요? (키워드: 캡슐화, 비캡슐화)
    - 캡슐화: 상위 계층에서 하위 계층으로 데이터를 전달할 때, 각 계층에서 필요한 헤더 정보를 추가하여 데이터를 패킷 형태로 만듭니다.
    - 비캡슐화: 하위 계층에서 상위 계층으로 데이터를 전달할 때, 각 계층에서 헤더 정보를 제거하여 원래의 데이터로 복원합니다.
  - 네트워크에서 PDU에 대해 알고 있나요?
    - PDU(Protocol Data Unit)는 네트워크 통신에서 각 계층에서 처리되는 데이터 단위를 의미합니다. 예를 들어, 전송 계층에서는 세그먼트, 인터넷 계층에서는 패킷, 네트워크 인터페이스 계층에서는 프레임이 PDU에 해당합니다.
  - 로드밸런서란 무엇인가요?
    - 로드밸런서는 네트워크 트래픽을 여러 서버에 분산시켜 서버의 부하를 균등하게 유지하고, 시스템의 가용성과 성능을 향상시키는 장치나 소프트웨어입니다.
  - ARP란 무엇인가요?
    - ARP(Address Resolution Protocol)는 IP 주소를 물리적 MAC 주소로 변환하는 프로토콜입니다. 네트워크 내에서 통신을 위해 필요한 주소 정보를 제공합니다.
  - MAC 주소란 무엇인가요?
    - MAC(Media Access Control) 주소는 네트워크 인터페이스 카드(NIC)에 할당된 고유한 식별자입니다. 48비트 길이의 16진수로 표현되며, 네트워크 내에서 장치를 식별하는 데 사용됩니다.
  - NAT란 무엇인가요?
    - NAT(Network Address Translation)는 사설 네트워크의 IP 주소를 공인 IP 주소로 변환하여 인터넷과 통신할 수 있도록 하는 기술입니다. 이를 통해 여러 장치가 하나의 공인 IP 주소를 공유할 수 있습니다.

- `HTTP 🔥`
  - HTTP란 뭔가요?
    - HTTP(HyperText Transfer Protocol)는 웹에서 클라이언트와 서버 간에 데이터(HTML, Javascript, 이미지 등)를 주고받기 위한 프로토콜 입니다.
  - HTTP 프로토콜의 가장 큰 특징은 뭔가요?
    - 무상태성 (Stateless): 각 요청이 독립적으로 처리되며, 이전 요청의 상태를 기억하지 않습니다.
  - URL은 뭔가요?
    - URL(Uniform Resource Locator)은 웹에서 자원의 위치를 나타내는 문자열입니다.
  - HTTP/1.0 과 HTTP/1.1의 차이는 뭔가요?
    - HTTP/1.0은 각 요청마다 새로운 TCP 연결을 생성하지만, HTTP/1.1은 지속적인 연결을 지원하여 여러 요청을 하나의 연결로 처리할 수 있습니다.
  - HTTP/1.1 과 HTTP/2.0의 차이는 뭔가요?
    - HTTP/2.0은 멀티플렉싱, 헤더 압축, 서버 푸시 등의 기능을 도입하여 성능을 향상시켰습니다. 반면, HTTP/1.1은 이러한 기능이 없습니다.
  - HTTPS는 HTTP랑 뭐가 다른가요? 🔥🔥🔥🔥
    - HTTPS는 HTTP에 SSL/TLS 프로토콜을 추가하여 데이터 전송 시 암호화를 제공함으로써 보안을 강화한 프로토콜입니다.
    - SSL/TLS 🔥🔥
      - SSL(Secure Sockets Layer)과 TLS(Transport Layer Security)는 인터넷 통신에서 데이터의 기밀성, 무결성, 인증을 제공하는 암호화 프로토콜입니다.
    - OSI 7계층에서의 SSL/TLS 🔥
      - SSL/TLS는 OSI 모델의 전송 계층과 응용 계층 사이에서 작동하여 데이터의 암호화와 보안을 제공합니다.
    - HTTPS가 제공하는 세 가지 보안 특성 🔥🔥
      - 기밀성: 데이터가 암호화되어 중간에 가로채더라도 내용을 알 수 없습니다.
      - 무결성: 데이터가 전송 중에 변경되지 않았음을 보장합니다.
      - 인증: 서버와 클라이언트가 서로의 신원을 확인할 수 있습니다.
  - 암호화 키의 종류에 대해 아나요? 🔥🔥🔥
    - 비대칭키(공개키/개인키) 암호화 🔥🔥
      - 공개키 암호화는 두 개의 키(공개키와 개인키)를 사용하여 데이터를 암호화하고 복호화하는 방식입니다. 공개키는 누구나 접근할 수 있지만, 개인키는 소유자만이 알고 있어야 합니다.
    - 대칭키와 비대칭키의 혼합 사용 🔥🔥
      - 혼합 사용은 비대칭키 암호화를 통해 대칭키를 안전하게 교환한 후, 대칭키를 사용하여 실제 데이터 전송을 암호화하는 방식입니다. 이는 비대칭키의 보안성과 대칭키의 속도 장점을 결합합니다.
    - TLS 핸드셰이크 과정 🔥🔥
      - 클라이언트 헬로우: 클라이언트가 서버에 연결 요청과 함께 지원하는 암호화 알고리즘 등을 전송합니다.
      - 서버 헬로우: 서버가 클라이언트의 요청을 수락하고, 선택된 암호화 알고리즘과 서버 인증서를 전송합니다.
      - 키 교환: 클라이언트와 서버가 대칭키를 생성하고 교환합니다.
      - 세션 암호화 설정: 양측이 대칭키를 사용하여 이후의 통신을 암호화하기로 합의합니다.
      - 완료: 핸드셰이크가 완료되면, 안전한 통신이 시작됩니다.
    - TLS 1.3 핸드셰이크 (1-RTT) 🔥🔥
      - TLS 1.3은 핸드셰이크 과정을 단순화하여 1-RTT(Round Trip Time)로 줄였습니다. 클라이언트가 초기 메시지에 필요한 정보를 모두 포함하여 서버에 전송하고, 서버는 즉시 응답을 보내어 핸드셰이크를 완료합니다.
    - TLS 1.2와의 차이점 🔥
      - TLS 1.3은 불필요한 암호화 알고리즘과 기능을 제거하여 보안을 강화하고, 핸드셰이크 과정을 단축하여 성능을 향상시켰습니다.
  - HTTPS 인증서 발급 과정에 대해 아나요? 🔥🔥
    - 인증서 서명 요청(CSR) 생성: 서버 관리자가 공개키와 함께 CSR을 생성합니다.
    - 인증 기관(CA)에 제출: CSR을 신뢰할 수 있는 인증 기관에 제출하여 인증서를 요청합니다.
    - 인증서 발급: CA는 CSR을 검토하고, 서버의 신원을 확인한 후 인증서를 발급합니다.
    - 서버에 설치: 발급된 인증서를 서버에 설치하여 HTTPS 통신을 설정합니다.
    - 클라이언트 검증: 클라이언트는 서버의 인증서를 검증하여 신뢰성을 확인합니다.

- `운영체제 🔥`
  - 운영체제란 무엇인가요?
  - 운영체제는 컴퓨터 하드웨어와 소프트웨어 자원을 관리하고, 컴퓨터 프로그램이 원활하게 실행될 수 있도록 지원하는 시스템 소프트웨어입니다.
  - 운영체제는 어떤 역할을 하나요?
  - 운영체제는 하드웨어 자원 관리, 파일 시스템 관리, 프로세스 관리, 메모리 관리, 보안 관리 등 다양한 역할을 수행합니다.
  - 운영체제는 어떤 구조로 이루어져 있나요?
    - 운영체제는 커널, 시스템 라이브러리, 시스템 유틸리티 등으로 구성됩니다. 커널은 하드웨어와 직접 상호작용하며, 시스템 라이브러리는 응용 프로그램이 운영체제 기능을 사용할 수 있도록 인터페이스를 제공합니다. 시스템 유틸리티는 운영체제의 다양한 기능을 관리하는 도구입니다.
  - 컴퓨터는 어떤 요소로 이루어져 있나요?
    - 컴퓨터는 중앙 처리 장치(CPU), 메모리, 저장 장치, 입력 장치, 출력 장치 등으로 이루어져 있습니다.
  - CPU는 어떤 요소로 이루어져 있나요?
    - CPU는 제어 유닛, 산술 논리 유닛(ALU), 레지스터 등으로 구성됩니다. 제어 유닛은 명령어를 해석하고 실행을 제어하며, ALU는 산술 및 논리 연산을 수행합니다. 레지스터는 데이터를 임시로 저장하는 고속 메모리입니다.
  - CPU는 어떻게 동작하나요?
    - CPU는 명령어 사이클을 통해 동작합니다. 명령어 사이클은 명령어 인출, 명령어 해석, 명령어 실행의 세 단계로 이루어집니다. CPU는 메모리에서 명령어를 인출하여 제어 유닛이 해석하고, ALU가 필요한 연산을 수행한 후 결과를 레지스터에 저장합니다.
  - 메모리란 무엇인가요?
  - 메모리는 컴퓨터에서 데이터를 저장하고 접근하는 장치입니다. 주로 RAM(랜덤 액세스 메모리)과 ROM(읽기 전용 메모리)으로 구분됩니다. RAM은 프로그램 실행 중에 데이터를 임시로 저장하는 반면, ROM은 시스템 부팅과 같은 중요한 정보를 영구적으로 저장합니다.
  - 메모리는 어떤 계층 구조로 이루어져 있나요?
    - 메모리는 레지스터, 캐시, 주 메모리(RAM), 보조 저장 장치(하드 드라이브) 등으로 계층 구조를 이루고 있습니다. 레지스터는 CPU 내부에 위치하여 가장 빠른 접근 속도를 제공하며, 캐시는 CPU와 RAM 사이에서 데이터를 임시로 저장하여 성능을 향상시킵니다. 주 메모리는 프로그램 실행에 필요한 데이터를 저장하고, 보조 저장 장치는 대용량 데이터를 영구적으로 저장합니다.
  - 메모리는 어떤 방식으로 데이터를 관리하나요?
    - 메모리는 가상 메모리와 물리적 메모리로 나뉘며, 운영체제는 가상 메모리를 통해 프로그램이 필요한 만큼의 메모리를 사용할 수 있도록 관리합니다. 페이지 교체 알고리즘을 사용하여 메모리 부족 상황에서 데이터를 디스크로 이동시키거나 불러오는 방식으로 데이터를 관리합니다.
